# summary.md

## 这次 MVP 的主线

我想先做一个个人网站 MVP，用它从 0 到 1 走一遍网页开发流程。当前不是追求网站多复杂，而是先用原生 HTML、CSS、JavaScript 理解结构、样式、交互和部署闭环。第一版分成顶部介绍和技术栈、中间项目入口、底部博客和思考。

## 我这次真正学到的东西

- 网页先拆结构，再做样式，最后加交互。
- HTML 是页面树，CSS 是展示和布局规则，JS 是行为。
- MVP 不需要一开始引入框架、后台、登录、数据库。
- Git 保存学习阶段，GitHub 做远程同步。
- 部署不该等到最后，最小版本要尽早跑通闭环。

## 我原来的误区

我原来以为部署是很后面的事情，要等网站写得差不多再做。现在理解是：部署可以很丑，但要尽早跑通链路，不是一口气把所有功能写完。

## AI 在这次学习中的作用

AI 帮我把模糊需求拆成小任务：HTML 骨架、CSS 布局、JS 交互。它不替我学习，我要能看懂它写在哪里、为什么这么写。

## 1 分钟口述稿

这次我做个人网站 MVP，不是为了立刻做成熟网站，而是想从 HTML、CSS、JavaScript 开始，亲手走一遍网页从本地到公网的流程。我没有一开始用 Next.js、React 或模板，而是先把页面拆成顶部介绍和技术栈、中间项目入口、底部博客和思考。过程中我理解到：HTML 负责结构，CSS 负责样式和布局，JavaScript 后面负责交互。以前我觉得部署是最后一步，现在发现应该先做最小版本，尽早跑通闭环。AI 主要帮我拆任务和解释代码，而不是直接生成一个我看不懂的网站。
